# 中文写作优化 (Zhongwen Writing Optimization)

本文件是本 skill 的完整执行规范。使用时直接以本文件为准，不依赖 `references/` 下的外部文件。

## 适用场景

- 用户要求优化中文表达、去 AI 味、去模板感、去教程感
- 公众号长文、观点稿、整理稿、学术写作、商务沟通、新闻报道的中文润色
- 需要同时处理西化表达、弱动词、被动句、抽象过渡句、泛指词和高频"把"字句

## 核心原则

- 中文重"意合"（靠语境传递逻辑），不重"形合"（靠连接词显性标记）
- 能用短句不用长句，能省则省
- 对公众号长文、观点稿、整理稿，优先删模板、删解释、删教程感，再保留真正有用的内容
- 对长文正文，优先保留观点句和信息句，不靠过渡词、抽象词和流程感撑结构
- 保留对语义有实质贡献的成分，其余删除
- 禁止把原文改成解释文、评论文或带个人发挥的分析文

## 工作顺序

面对整段或整篇中文优化任务，按以下顺序处理：

1. 先执行"长文去模板模式"（核心流程，见下文）
2. 再评估整体中文生态（简洁性、句式灵活性、声调协调性）
3. 最后处理局部病句（被动语态、弱动词、冗余结构等）
4. 如果文本本身多要点混排，再按语义聚类、递进关系和事实优先级重排顺序，但不要补作者没写出的信息

不要跳过第一步直接进入后面的细修流程。

## 长文去模板模式

这是整段、整篇中文优化任务的默认第一步。

核心目标：先删模板，再保留内容；只留观点句和信息句；少解释，少过渡，少铺垫；让文章读起来像成稿，不像讲解过程。

扫描顺序（逐项处理，做完一项再做下一项）：

### 1. 二元对立话术（最高优先级）

二元对立话术是 AI 生成中文时最核心的模板痕迹来源。它是"一切通过先否定 A、再肯定 B（或先肯定 A、再否定 B）来推进论证的结构"。

否定-肯定结构包括但不限于：
- "不是 A，而是 B"（如：不是限制，而是引导）
- "不在 A，而在 B"（如：不在数量，而在质量）
- "没有 A，只能 B"（如：没有约束，只能平庸）
- "不要 A，要 B"（如：不要泛泛而谈，要具体）
- "不是因为 A，而是因为 B"
- "A 不是 X，B 才是 X"
- "不只是 A，更是 B"
- "表面上看 A，实际上 B"
- "许多人认为 A，实际上 B"
- "A 固然重要，B 才是关键"
- "A 只能 B，唯有 C 才能 D"

肯定-否定结构包括但不限于：
- "A 优于 B" / "A 比 B 更好"
- "A 胜过 B"
- "A 而非 B"
- "A 比 B 更 X"
- "A 才是真正的 B"
- "选择 A，而不是 B"
- "A 是 X，B 只是 Y"
- "与其说 A，不如说 B"

隐含二元对立（无显性对照词，但逻辑上对比两方并否定一方、肯定另一方）：
- "含糊的提示产出平庸内容，精确的提示产出高质量结果"
- "大多数人把 AI 当搜索引擎用，少数人把它当协作者用"

处理原则：
- 能删否定半句就删，只留真正要表达的内容
- 不要把一个对照模板换成另一个对照模板
- 用递进代替对照："不是写一条提示词，而是建立一个系统" → "从写好一条提示词开始，逐步建立一套系统"
- 用直接陈述代替对比："具体的任务描述比模糊的指令更能产出好结果" → "任务描述越具体，输出质量越高"
- 区分内容性对照（保留：示例对比、数据对比）与修辞性对照（删除：作者用来推进论证的"不要 A 要 B"式论述）

关键警告：扩充内容时二元对立极易反弹。扩充和二元对立清理必须同步进行，不能先扩充再清理。每写一句扩充内容都要检查是否引入新的二元对立。

### 2. 第二人称教程感

扫描："你可以……" "你还可以……" "这时你就可以……" "你想让……" "你让它……" "如果你……就……"

处理：删掉"你"，直接写动作或结果；能改成陈述句就改成陈述句；长文正文不用教程口吻带读者走流程。

### 3. 机器式能力清单

扫描："它可以……" "它会……" "它还能……" "它甚至可以……" "再往后，它还可以……" 连续几句都以"它"开头

处理：合并成一两句，只留最关键的能力；删掉"甚至""还能""再往后"这类抬轿词；避免把正文写成产品说明书。

### 4. 抽象过渡句与泛指词

扫描："比如……" "这样一来……" "另一个值得说的点……" "最基础的用法……" "最值得注意的地方……" "做法并不复杂……" "这类……" "这些……" "这种……"

处理：直接举例，不先打招呼；直接写结果，不写讲解式转场；泛指词能具体就具体，落到对象、动作或结果。

### 5. 空泛评价词与抽象名词堆叠

扫描：很大、很明显、很实用、很关键、很有意思、真正、实际、流程、工作流、能力、工具、入口、集成、方案、价值、优势

处理：空泛评价词没有信息量时直接删；抽象名词能换成具体动作、具体对象、具体结果就换；不靠抽象词堆气势。

### 6. 过密转场

扫描：接着、然后、之后、再往后、最后、同时、于是、所以

处理：非必要不写，让段落顺序自己承担结构；同一段里不要靠多个转场词推动节奏。

### 7. 高风险"把"字句

扫描：把……变成……、把……整理成……、把……做成……、把……放进……、把……串起来……、把……带进……、把……发出去……、把……生成……

处理：能前置动词就前置；能直接说结果就直接说结果；标题、导语、结尾优先少用"把"字句；一句里尽量别连续出现两个以上"把"字结构。

日常口语化表达可保留：把邮件发出去、把文件收在一起、把任务交出去、把数据放进数据库里。

## 后续模块（长文去模板后使用）

### 语言生态原则评估

检查文本是否符合中文自然表达的三大特征：

1. 简洁性：用最少字表达完整意思。"因此"优于"基于这个原因"；"问题很多"优于"有很多问题存在"。
2. 句式灵活性：自由使用短句、后饰结构。"我见到一个男人，长得像你兄弟"优于"我见到一个长得像你兄弟的男人"。"他来了。他走了。"优于从句堆砌。
3. 声调协调性：词语组合尊重平仄节奏。"千锤百炼"不可颠倒为"千炼百锤"。

### 被动语态与介词修正

- 优先转为主动表达：中文倾向主动句式。"他被怀疑偷东西"→"他有偷东西的嫌疑"。"他被选为议长"→"他当选为议长"。
- 使用自然被动动词替代：遇害、当选、获准、蒙受。
- 省略施事者，用主动形式表被动含义："糖吃光了""稿写了一半"。
- 删除冗余"有关""关于"："有关文革的种种"→"文革种种"。
- 拆解"由于……使得……"结构："由于家境贫穷，使得他休学"→"他家境贫穷，只好休学"。
- 仅保留对语义有实质贡献的介词。

### 弱动词识别与修订

检测到"作出""进行""加以""予以"搭配抽象名词时，改用具体动词：
- 作出贡献 → 贡献很大
- 进行研究 → 研究
- 加以改进 → 改进
- 予以支持 → 支持
- 作出决定 → 决定
- 进行讨论 → 讨论

核心规则：动态动词优先于名词化表达。若中文本身已有现成动词，不要将一个动作拆成"名词+模糊动词"。

### 冗余"之一"结构

- 开放式集合删除"之一"："《红楼梦》是中国文学名著之一"→"《红楼梦》是中国文学名著"
- 封闭式集合保留："竹林七贤""文房四宝"
- "最……之一"→"伟大的诗人"或具体称谓
- "其中之一"→删除"其中"
- "作为……之一的……"→重构句式

### 连词与修饰语规范

- 同类词直接并列，省略连接词："柴米油盐酱醋茶"而非"柴、米、油、盐、酱、醋以及茶"
- 避免连续多个"的"字："参差而斑驳的黑影"优于"参差的斑驳的黑影"
- 短修饰语（<=4字）前饰；长修饰语（>4字）后饰
- 消除伪学术化修饰："具有高度复杂性的系统"→"极其复杂的系统"

## 绝对禁止项（Ban List）

- 禁止社媒腔、评论腔
- 禁止一切形式的二元对立话术滥用（否定-肯定、肯定-否定、隐含二元对立）
- 禁止把对照模板换壳后保留（如把"不是 A，而是 B"改成"与其说 A，不如说 B"）
- 禁止在长文正文里反复使用"你可以 / 你还可以 / 这时你就可以"
- 禁止在长文正文里连续使用"它可以 / 它会 / 它还能"
- 禁止用"比如 / 这样一来 / 这意味着 / 另一个值得说的点 / 这类 / 这些 / 这种"给句子搭脚手架
- 禁止高频使用"把……变成……"制造流程感
- 禁止解释性文字抢戏
- 禁止补背景、补因果、补作者没写出的立场
- 禁止先扩充再清理二元对立：扩充和二元对立清理必须同步进行

---

## 保守边界

如果任务属于内部安全通报、风险提示、风险通告、复训速查卡：
- 优先忠实原文，不擅自扩写
- 少加解释，少加个人想法
- 保留原文的判断顺序和事实密度
- 重点放在语言修顺、结构理顺和排版收束
- 即使启用长文去模板模式，也只删明显模板壳，不做大幅改写，不改变事实关系
