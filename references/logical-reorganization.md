# 多要点文本的逻辑重组

## 何时使用

- 用户提交包含多条要求/提示/建议的文本，但顺序混乱、内容交叉重叠
- 各条目之间缺乏清晰的逻辑递进关系（如时间线、因果链、层次结构）
- 用户明确反馈"顺序是乱的""不符合正常逻辑"等
- 单条内容内部混杂不同层面的信息（如把航前准备和滑行操作混在一起）

## 核心方法论：打散 → 归类 → 排序 → 定界

### 第一步：打散（解构）

将原有条目全部拆解为最小语义单元，不再受原条目边界约束。

**示例**：
- 原第1点"结合机场通告、平面图、使用跑道和停机位信息提前熟悉滑行路线及机场特殊限制" → 拆为"熟悉机场资料""提前熟悉滑行路线""了解机场特殊限制"
- 原第4点"下降准备阶段，应对脱离方向、滑行路线提前做好预案" → 拆为"下降前做预案""明确脱离方向""明确滑行路线"

### 第二步：归类（聚类）

按**内在逻辑主题**将打散后的单元聚类，而非保留原有的条目框架。

**归类维度**：
- **时间维度**：事前 / 事中 / 事后
- **层次维度**：预案层面 / 准备层面 / 执行层面 / 应变层面
- **因果维度**：前提条件 / 核心操作 / 兜底保障

**示例**（航空滑行提示）：
| 主题 | 归入内容 |
|------|----------|
| 事前知路 | 航前熟悉资料 + 下降前做预案 |
| 严谨听令 | 确认指令 + 复诵 + 未获令不动 |
| 规范操作 | 分工协作 + 减速防错 + 观察标识 |
| 底线应变 | 存疑即停 + 及时证实 |

### 第三步：排序（定序）

为归类后的主题组确定正确顺序，遵循以下优先级：

1. **时间优先**：按事件发生的自然时序排列（先发生的在前）
2. **因果优先**：按条件→行动→结果的因果链排列
3. **层次优先**：按宏观→微观、战略→战术、准备→执行的层次排列

**通用排序模式**：
```
事前预案 / 提前准备
    ↓
接收指令 / 确认信息
    ↓
执行操作 / 规范行为
    ↓
底线应变 / 应急处理
```

### 第四步：定界（措辞）

为每条重新定界，确保：
- **边界清晰**：本条只讲一个层面，不越界混入其他层面的内容
- **措辞精炼**：用最小语义单元概括本条核心，避免重复其他条已讲的内容
- **逻辑衔接**：相邻条目之间形成自然的递进或承接关系

**示例**：
- 不佳："落实进场简令"（太笼统，简令内容分散在原第4点各处）
- 正确："提前熟悉路线"（只聚焦"事前知路"这一层，"听令""复诵"等已移至其他条）

## 执行步骤

1. 通读原文所有条目，标记每个条目的核心语义单元
2. 按时间/因果/层次维度将语义单元重新归类，列出归类表
3. 为归类后的主题确定正确顺序，说明排序依据
4. 逐条重新措辞，确保单条边界清晰、条间衔接自然
5. 输出重组后的全文，附逻辑链条说明

## 常见陷阱

| 陷阱 | 表现 | 对策 |
|------|------|------|
| 伪归类 | 只是调换了原条目的顺序，没有真正打散重组 | 必须拆解到最小语义单元 |
| 内容残留 | 某条的核心内容被拆分后，原条目仍保留空壳 | 彻底清空原框架，全部重排 |
| 边界模糊 | 新条目之间仍有内容交叉 | 每条只讲一个层面，重复内容合并到最恰当的位置 |
| 逻辑跳跃 | 相邻条目之间缺乏自然的递进关系 | 检查排序是否符合时间/因果/层次原则 |
